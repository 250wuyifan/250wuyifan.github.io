一个详细解释

这行代码定义了一个函数指针 pVirtualAlloc，并将其初始化为指向 Windows API 函数 VirtualAlloc的地址。让我们逐步解析：

  1. LPVOID(WINAPI * pVirtualAlloc)(LPVOID, SIZE_T, DWORD, DWORD)定义了一个函数指针 pVirtualAlloc该函数指针类型为：返回 LPVOID，调用约定为 WINAPI（即 __stdcall），接受四个参数（LPVOID, SIZE_T, DWORD, DWORD）
  2. (LPVOID(WINAPI*)(LPVOID, SIZE_T, DWORD, DWORD))这是一个类型转换，将后面的表达式转换为上述函数指针类型
  3. GetProcAddress(hKernel32, "VirtualAlloc")从已加载的 kernel32.dll 模块（通过句柄 hKernel32）中获取 VirtualAlloc函数的地址

整体作用：

  • 这行代码动态获取 VirtualAlloc函数的地址，并将其赋值给函数指针 pVirtualAlloc
  • 之后可以通过 pVirtualAlloc来调用 VirtualAlloc函数

get_ntdll_base_address

这个函数 get_ntdll_base_address的目的是 动态猜测 ntdll.dll在内存中的基地址，而不依赖传统的 Windows API（如 GetModuleHandle）。它通过尝试不同的内存地址组合，并结合自定义的 CustomGetProcAddress来验证是否找到了正确的 ntdll.dll基址。

函数功能

  • 输入：一个表示内存地址的字符串（如 "7FFA00000000"）。
  • 输出：返回 ntdll.dll在内存中的基地址（void*指针）。
  • 核心逻辑：解析输入地址的格式（拆分为前缀和最后6位十六进制值）。尝试调整最后6位中的部分比特（高位 nibble），生成可能的 ntdll.dll基地址。用 CustomGetProcAddress检查这些地址是否能成功解析 NtClose函数。如果找到有效的基地址，立即返回；否则继续尝试其他组合。

详细解析

输入地址的拆分

假设输入地址是 "7FFA12345678"：

  • 前缀："7FFA12"（前 len-6个字符）。
  • 最后6位："345678"（转换为 unsigned int值 0x345678）。

分解最后6位的比特

将 0x345678分解为：

  • 最高4位（nibble）：3（(last6_val >> 20) & 0xF）。
  • 次高4位：4（(last6_val >> 16) & 0xF）。
  • 低16位：0x5678（暂时忽略，后续设为 0）。

** 尝试地址组合**

第一轮尝试：固定最高4位，遍历次高4位
  • 保持 first_nibble = 0x3，遍历 second_nibble从 0x0到 0xF。
  • 生成地址格式：前缀 + (first_nibble << 20 | second_nibble << 16 | 0)。例如："7FFA12000000"、"7FFA12100000"、…、"7FFA12F00000"。
第二轮尝试：最高4位减1，再次遍历次高4位
  • 如果第一轮失败，尝试 first_nibble = 0x3 - 1 = 0x2。
  • 生成地址格式："7FFA02000000"、"7FFA02100000"、…、"7FFA02F00000"。

验证基地址

  • 对每个生成的地址，调用 CustomGetProcAddress(base_address_test, "NtClose")。
  • 如果返回非 NULL，说明该地址是有效的 ntdll.dll基址（因为 NtClose是 ntdll.dll的导出函数）。

CustomGetModuleHandle

这个函数 CustomGetModuleHandle是一个 手动实现 的 GetModuleHandle，用于在 远程进程 中查找指定 DLL（如 ntdll.dll）的基地址。它通过直接解析目标进程的 PEB（Process Environment Block） 和 LDR（Loader）数据结构 来定位 DLL，而不依赖 Windows API（如 GetModuleHandle或 EnumProcessModules）。

函数功能

  • 输入：hProcess：目标进程的句柄（需有 PROCESS_QUERY_INFORMATION权限）。dll_name：要查找的 DLL 名称（如 "ntdll.dll"）。
  • 输出：返回 DLL 在目标进程中的基地址（uintptr_t）。如果找不到，返回 0。
