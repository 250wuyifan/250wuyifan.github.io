Windows 线程池

Windows 线程池机制显著简化了程序员的线程管理。异步交互和性能管理等任务通过重用现有线程在后台得到解决，从而减少了创建和终止线程所需的 CPU 成本。同时，Windows 线程池还解决了与线程队列相关的问题以及无数可能让程序员抓狂的细微差别。

管理线程池的代码大部分位于用户模式（），只有一小部分位于内核；因此，没有必要在内核和用户空间之间频繁进行上下文切换而浪费资源。<font style="color:rgb(0, 0, 0);background-color:rgb(248, 248, 248);">ntdll.</font><font style="color:rgb(0, 0, 0);background-color:rgb(248, 248, 248);">dll</font>

Windows线程池的主要元素如下：

  • 管理新线程的创建和删除的 Worker Factory；
  • 工作队列：
    • 任务队列负责调度传入线程池的任务。这些任务可以很快（即在短时间内执行），也可以执行时间较长。线程池会自动扩展；如果任务数量较多，线程池可以创建更多线程来应对负载。
    • I/O完成队列执行与I/O操作相关的任务（例如文件操作或网络请求）；
    • 定时器队列使得按照一定的时间间隔或者在指定的时间执行任务成为可能。

从 PoolParty 开发角度来看，上面列出的线程池元素是最受关注的。

需要注意的是，Windows 环境中运行的所有进程默认都使用线程池。为了通过实验验证这一点，您可以启动 Process Explorer，选择任意进程，然后转到“句柄”选项卡。

捕获目标句柄

与线程池元素的交互是通过句柄实现的。因此，要访问目标进程的工作工厂，必须找到它的句柄并使用 WinAPI<font style="color:rgb(0, 0, 0);background-color:rgb(248, 248, 248);">DuplicateHandle</font>函数捕获它。

线程池句柄分为几种类型。如果需要访问工作器工厂， <font style="color:rgb(0, 0, 0);background-color:rgb(248, 248, 248);">TpWorkerFactory</font>则应使用对象类型为 的句柄；要访问计时器队列，则应使用类型的句柄<font style="color:rgb(0, 0, 0);background-color:rgb(248, 248, 248);">IRTimer</font>；要访问 I/O 队列，则应使用<font style="color:rgb(0, 0, 0);background-color:rgb(248, 248, 248);">IoCompletion</font>类型的句柄。由于您要攻击工作器工厂，因此必须查找 <font style="color:rgb(0, 0, 0);background-color:rgb(248, 248, 248);">TpWorkerFactory</font>对象管理器类型。
