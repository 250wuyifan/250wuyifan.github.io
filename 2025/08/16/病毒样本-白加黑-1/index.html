一次针对白加黑的快速样本分析

首先在卡饭社区看到了这个白加黑的程序，于是下载下来分析。

分析流程

下载之后把压缩文件解压之后，立马有一个dll文件被杀，发现此文件无签名，初步判断这个dll就是黑dll

拖入ida分析dll，发现一个地方使用了当前目录下的Plungneackprog.ehra，

这个地方函数

这个函数里面大部分用上了以下两个函数，从代码来看，sub_10001B43是一个 API 哈希解析函数，而 sub_10001BC3是它的 哈希计算函数。这两个函数通常用于 动态获取 API 地址（类似 GetProcAddress），但使用了 自定义哈希算法 来规避静态分析。

完整如下

进行猜测，Plungneackprog.ehra存储加密的shellcode，然后通过dll进行解密shellcode，加载到内存中或者把shellcode释放为一个exe。

进行动态调试，使用process monitor进行监控主程序。在最后发现了执行了一个exe，一起进行监控

进行文件搜索此文件在C:\Users\CHANGMEN\AppData\Local\Temp\bin目录下，值得怀疑，看之后的操作创建了很多的temp文件，最显眼的就是这个网络的连接情况。

微步情报看一下

仔细看了一下这文件乱看，浏览我全盘文件，试图找敏感文件上传到刚刚的网络地址了。

试图去找一下关于网络通讯的操作，拖入ida，从导入表未找到关于网络通信的函数。单独启动这个poasm.exe是没有这么多浏览各种文件的操作的，只有启动setup.exe的时候才会有连接这个网络地址的操作。所以还是恶意的dll文件使用此poasm.exe文件的时候进行传入了参数进行操作的，目前能力有限，分析到此。
