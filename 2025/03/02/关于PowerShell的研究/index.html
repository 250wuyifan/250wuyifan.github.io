关于PowerShell的研究

什么是 PowerShell？

PowerShell 是微软为系统管理员和高级用户提供的多功能工具，常被称为“瑞士军刀”。它于 2006 年推出，融合了强大的命令行 shell 和脚本语言功能。

PowerShell 的独特之处在于：

  1. 面向对象：PowerShell 不仅能处理文本，还能处理结构化数据，使得在命令之间操作和传输复杂信息变得轻松自如。
  2. 基于 .NET 构建：这一点让 PowerShell 能够深入访问 Windows 内部，具有极强的系统管理能力。
  3. 跨平台支持：虽然 PowerShell 最初仅适用于 Windows，但现在它也可以在 Linux 和 macOS 上运行。
  4. 高效的管理工具：无论您是在管理单一笔记本电脑还是一组云服务器，PowerShell 都能帮助您更快速、高效地完成任务。因此，对于任何专注于 Windows 管理或 DevOps 的用户来说，它都是首选工具。

为什么红队喜欢 PowerShell

PowerShell 已经稳固地成为红队和渗透测试的核心工具，原因如下：

  1. 预装在 Windows 上：这意味着攻击者无需额外安装工具，减少了被检测的风险。
  2. 深度集成 Windows 和 .NET：攻击者可以利用这一点来窃取凭证、在网络中横向移动或提取敏感数据。
  3. 全面支持攻击过程：从发现目标到保持访问，PowerShell 都能处理攻击的各个步骤。
  4. 脚本隐藏和混淆：攻击者能够隐藏或扰乱脚本，使得防御者更难检测到它们。
  5. 与云平台兼容：PowerShell 能与 Azure 和 AWS 等云平台协同工作，适应现代攻击需求。

总之，PowerShell 不仅仅是一个工具，它已经成为应对现代攻击面不可或缺的强大盟友。

PowerShell 及其内存加载功能

PowerShell 的内存执行功能是红队梦寐以求的强大工具，原因如下：

  1. 代码直接在 RAM 中运行：这样代码不会在磁盘上留下任何痕迹，使得防病毒软件和其他安全工具难以检测到。
  2. 动态生成和混淆代码：攻击者可以根据不同环境动态生成、修改或混淆代码，从而快速绕过安全防护。
  3. 全框架和工具加载到内存：整个攻击框架和复杂工具都可以加载到内存中，攻击者无需在系统中安装任何软件即可使用完整功能。
  4. 低调且高效：一旦进入系统，内存执行可以支持从凭证收集到横向移动的所有攻击活动，同时保持低调，减少被发现的风险。

Invoke-Expression Cmdlet — 启用内存执行

Invoke-Expression（IEX）允许将字符串作为 PowerShell 命令在当前会话中执行，是实现内存执行的关键工具。

此命令运行“Get-Process”并返回活动进程列表。Invoke-Expression与其他 cmdlet 或 .NET 方法结合使用时功能更加强大，可实现复杂的内存中命令执行。
