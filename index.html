样本分析

静态分析

我们可以使用UNIX/Windows的FILE程序来看文件类型，输出“PE32可执行文件（GUI）Intel 80386 Mono/.Net程序集，用于MS Windows”表明我们的样本是.NET程序集

.NET程序集是一种通过编译.NET语言（如C#）代码生成的DLL或EXE文件。不过，它并不包含本地x86/x64机器码，而是使用MSIL（微软中间语言）字节码。这些MSIL由.NET运行时执行，并在运行时将其编译为本地代码。

分析这类文件的利器当属dnSpy——这款集.NET调试器、汇编编辑器和反编译器功能于一身的工具。当我们把示例文件加载到dnSpy时，会立即发现某些函数和对象的名称变成了\u0001、\u0002等乱码字符，而非人类可读的名称。这表明该示例文件已被混淆处理。需要说明的是，文件混淆本身并不具有恶意属性，企业通常会通过混淆技术来保护知识产权。不过，这种现象仍值得我们特别留意。

我们将使用de4dot工具，该工具用于检测常见的混淆器并清理.NET二进制文件。通过将二进制文件拖放到de4dot.exe中。

目前我们已完成对样本的初步分类。通过文件分析、dnSpy和de4dot工具，我们确认该样本为.NET程序集，检测到其存在混淆处理，并完成了反混淆操作。在完成反混淆处理后，现在可以开始深入分析这段代码了。

调试本地代码的执行

开头先介绍一下托管代码（managed code）与非托管代码（unmanaged code）的概念。微软提供的这份资料是一个很好的入门指南，但简单来说：

•托管代码（通常用 C# 等语言编写）运行在公共语言运行时（CLR）环境中，由 CLR 负责内存管理和安全性。

•非托管代码（通常用 C 或 C++ 编写）则直接由操作系统执行，因此需要程序手动管理内存。

之所以提到这一点，是因为该样本的执行已从托管代码切换到了非托管代码。在上一节中，我们观察到样本在运行时环境中下载了一个有效负载（payload），将其写入内存，并将指向该内存的指针转换为委托（delegate）后调用。

在此之前，我们还能用 dnSpy 单步调试托管代码，但由于执行跳转到了非托管代码，我们失去了对样本执行流的可见性，也无法再用 dnSpy 的调试器控制其执行。接下来，我们需要想办法使用原生调试器（如 x64dbg）附加到进程，以便分析shellcode行为。

部分总结2 禁用安全功能asmi和etw

总而言之，恶意软件通过以下操作禁用了几个对安全至关重要的重要Windows API函数：

  3. 调用VirtualProtect将内存保护设置为RWX

  5. 调用VirtualProtect将内存保护设置为R-X

这种内存修补方法通常被恶意行为者或红队用来绕过ETW和AMSI等组件。
我们还观察到恶意软件使用相对于PC的寻址来定位资源，例如用于重写AmsiScanBuffer、AmsiScanString和EtwEventWrite的伪代码。

关于PowerShell的研究

什么是 PowerShell？

PowerShell 是微软为系统管理员和高级用户提供的多功能工具，常被称为“瑞士军刀”。它于 2006 年推出，融合了强大的命令行 shell 和脚本语言功能。

PowerShell 的独特之处在于：

  1. 面向对象：PowerShell 不仅能处理文本，还能处理结构化数据，使得在命令之间操作和传输复杂信息变得轻松自如。
  2. 基于 .NET 构建：这一点让 PowerShell 能够深入访问 Windows 内部，具有极强的系统管理能力。
  3. 跨平台支持：虽然 PowerShell 最初仅适用于 Windows，但现在它也可以在 Linux 和 macOS 上运行。
  4. 高效的管理工具：无论您是在管理单一笔记本电脑还是一组云服务器，PowerShell 都能帮助您更快速、高效地完成任务。因此，对于任何专注于 Windows 管理或 DevOps 的用户来说，它都是首选工具。

为什么红队喜欢 PowerShell

PowerShell 已经稳固地成为红队和渗透测试的核心工具，原因如下：

  1. 预装在 Windows 上：这意味着攻击者无需额外安装工具，减少了被检测的风险。
  2. 深度集成 Windows 和 .NET：攻击者可以利用这一点来窃取凭证、在网络中横向移动或提取敏感数据。
  3. 全面支持攻击过程：从发现目标到保持访问，PowerShell 都能处理攻击的各个步骤。
  4. 脚本隐藏和混淆：攻击者能够隐藏或扰乱脚本，使得防御者更难检测到它们。
  5. 与云平台兼容：PowerShell 能与 Azure 和 AWS 等云平台协同工作，适应现代攻击需求。

总之，PowerShell 不仅仅是一个工具，它已经成为应对现代攻击面不可或缺的强大盟友。

PowerShell 及其内存加载功能

PowerShell 的内存执行功能是红队梦寐以求的强大工具，原因如下：

  1. 代码直接在 RAM 中运行：这样代码不会在磁盘上留下任何痕迹，使得防病毒软件和其他安全工具难以检测到。
  2. 动态生成和混淆代码：攻击者可以根据不同环境动态生成、修改或混淆代码，从而快速绕过安全防护。
  3. 全框架和工具加载到内存：整个攻击框架和复杂工具都可以加载到内存中，攻击者无需在系统中安装任何软件即可使用完整功能。
  4. 低调且高效：一旦进入系统，内存执行可以支持从凭证收集到横向移动的所有攻击活动，同时保持低调，减少被发现的风险。

Invoke-Expression Cmdlet — 启用内存执行

Invoke-Expression（IEX）允许将字符串作为 PowerShell 命令在当前会话中执行，是实现内存执行的关键工具。

此命令运行“Get-Process”并返回活动进程列表。Invoke-Expression与其他 cmdlet 或 .NET 方法结合使用时功能更加强大，可实现复杂的内存中命令执行。
