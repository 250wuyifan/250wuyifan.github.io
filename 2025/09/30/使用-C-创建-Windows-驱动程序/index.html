代码解析

1. 头文件 #include <ntddk.h>

  • ntddk.h是 Windows 驱动程序开发的核心头文件，包含：内核 API（如 DbgPrint）驱动数据结构（如 DRIVER_OBJECT）基本宏（如 UNREFERENCED_PARAMETER）

2. 卸载函数 DriverUnload

  • 作用：当驱动程序被卸载时（如 sc stop MyDriver），系统会调用这个函数。
  • **UNREFERENCED_PARAMETER**：告诉编译器 DriverObject参数虽然未被使用，但这是有意为之（避免警告）。
  • DbgPrint**：内核模式的 printf，但输出不会显示在控制台，需要用 **DebugView 或 WinDbg 查看。

3. 驱动入口点 DriverEntry

  • 作用：驱动加载时，Windows 内核会调用 DriverEntry（类似于 main()）。

  • 参数：DriverObject：代表驱动程序本身，用于设置回调函数（如卸载函数）。RegistryPath：驱动在注册表中的路径（这里未使用）。

  • 关键操作：

    1. 设置卸载回调：

      • 告诉 Windows 卸载时调用 DriverUnload。

    2. 打印日志：

      • 驱动加载时输出调试信息。

    3. 返回状态：

      • STATUS_SUCCESS（0）表示驱动加载成功。
